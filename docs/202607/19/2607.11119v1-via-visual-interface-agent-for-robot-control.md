## 摘要
机器人操作是一项复杂的任务，需要视觉理解、物理推理、规划和闭环控制。通用基础模型（FMs）在这些能力中的某些方面（尤其是视觉和推理）已经取得了显著进展。为了利用这一点构建通用型机器人策略，当前的方法通常涉及通过在机器人数据上进行微调，将现有的FMs转换为视觉-语言-动作（VLA）模型，以输出低级动作。然而，由于微调可用的数据和计算资源有限，VLA的规模往往比前沿FM小数个数量级，这反过来限制了它们的通用能力。受FMs通过视觉界面操作软件的能力日益增强的启发，我们探究这种相同的能力是否足以控制机器人。我们提出了VIA（面向机器人控制的视觉界面智能体），一个将机器人控制重新定义为智能体任务的框架：一个基于现成FM驱动的智能体通过基于浏览器的3D界面操作机械臂，具体方式为截取屏幕截图、发出直观命令、观察结果并调整。该智能体没有经过针对机器人的微调，也无法访问特权状态信息：它感知视觉输入，并通过一小套通用工具进行操作。VIA继承了智能体的通用推理、闭环错误恢复以及根据观察进行规划和重新规划的能力。它使用Claude Code和Codex在零样本条件下解决了一系列桌面操作任务。使用最强模型（Fable 5），它在三项LIBERO-Goal任务上达到了96.7%的成功率，在一项长视界彩虹组装任务上达到了100%的成功率。性能随着基础模型的规模和强度提升而提高。这些结果表明，前沿智能体已经具备的技能在提供合适界面时可以直接迁移到机器人控制：你的编码或计算机使用智能体，在某种意义上，其实也是一个隐藏的机器人控制智能体。

## Abstract
Robot manipulation is a complex task that requires visual understanding, physical reasoning, planning, and closed-loop control. General-purpose foundation models (FMs) have grown remarkably capable of some of these, especially vision and reasoning. To leverage this for generalist robot policies, current methods typically involve converting existing FMs into vision-language-action (VLA) models by fine-tuning on robot data to output low-level actions. However, VLAs are often orders of magnitude smaller than frontier FMs given the limited data and compute available for fine-tuning, which in turn limits their general capability. Inspired by the growing ability of FMs to operate software through visual interfaces, we ask whether that same competence suffices to control a robot. We present VIA (Visual Interface Agent for robot control), a framework that recasts robot control as an agentic task: an off-the-shelf FM-powered agent drives a manipulator through a browser-based 3D interface by taking screenshots, issuing intuitive commands, observing the outcome, and adjusting. The agent receives no robot-specific fine-tuning and no access to privileged state information: it perceives visual input and acts through a small set of general tools. VIA inherits the agent's general reasoning, closed-loop error recovery, and ability to plan and re-plan from what it observes. It solves a diverse suite of tabletop manipulation tasks zero-shot with both Claude Code and Codex. With the strongest model (Fable 5) it achieves 96.7% success on three LIBERO-Goal tasks and 100% on a long-horizon rainbow assembly task. Performance improves with the scale and strength of the underlying model. These results suggest that frontier agents already possess skills that transfer directly to robot control given the right interface: your coding or computer-use agent is, in a sense, secretly a robot-control agent.

## 1. 核心问题与整体含义（研究动机与背景）

- **背景**：机器人操作任务要求视觉理解、物理推理、规划与闭环控制。通用基础模型（FMs，如 GPT、Claude）已具备强大的视觉与推理能力。现有方法通常将 FM 微调为视觉-语言-动作（VLA）模型，让其直接输出低级动作。然而，由于缺乏大规模机器人数据和计算资源，微调后的 VLA 模型通常比前沿 FM 小几个数量级，导致其通用能力（如物理推理、长程规划）受限。
- **核心问题**：如何在不进行机器人专用微调的前提下，充分利用前沿 FM 的完整能力来控制机器人？
- **核心思想**：受 FM 通过视觉界面操作软件（如计算机使用）的启发，本文提出将机器人控制重新定义为**智能体任务**：一个现成的基础模型智能体通过浏览器中的 3D 视觉界面操控机械臂，像人类一样“截图—命令—观察—调整”。智能体无需任何机器人专用微调，也无需访问特权状态信息，仅凭视觉输入和少量通用工具完成任务。

## 2. 方法论：核心思想、关键技术细节

- **整体框架**：VIA 构建一个**浏览器端 3D 机器人控制界面**。该界面由多台 RGB-D 相机重建的 3D 点云构成主工作区，辅以第三人称视图和腕部视图。智能体通过截取界面截图感知环境，通过 MCP（Model Context Protocol）工具发出直观命令。
- **关键组件**：
  - **视觉界面**：类似 3D 设计软件（如 Blender），支持视角旋转、平移、缩放。智能体只能看到截图，无法直接读取原始点云向量或特权状态。
  - **MCP 工具集**（四类）：
    - **观察工具**：`screenshot`, `hover`, `gripper_get_pose`, `gripper_show_rotation_gizmo`, `camera_get_pose`。
    - **操作工具**：`gripper_teleport_via_click`, `gripper_drag`, `gripper_translate`, `gripper_advance_or_retreat`, `gripper_rotate`, `gripper_toggle`, `gripper_reset`。这些工具仅改变虚拟目标夹爪（蓝色网格），不移动真实机器人。
    - **相机工具**：`camera_orbit_via_key`, `camera_pan_via_key`, `camera_zoom`, `camera_reset`。
    - **执行工具**：`execute_waypoint`（将虚拟目标夹爪的位姿通过 PI 控制器传递给真实机器人），`end_episode`。
  - **智能体循环**：智能体通过循环“观察→思考→调用工具→获得新截图”完成多步骤任务。每个工具调用返回更新后的截图，实现闭环控制与错误恢复。无需额外的代码生成或感知基元。
- **关键技术细节**：
  - 智能体的操作与人类操作界面方式相同：通过点击、拖拽、键盘输入等方式调整虚拟目标夹爪的 6-DoF 位姿和夹爪开关状态，设置好“航点”（waypoint）后调用执行工具移动真实机器人。
  - 所有工具设计遵循**最小化**（每个工具只封装对应的人类操作）和**智能体人体工程学**（考虑智能体通过离散截图观察，提供如 `gripper_rotate(axis, angle)` 而非连续拖拽）。
  - 智能体仅使用一个简短系统提示（约 650 词），包含界面说明和通用操作建议（如分解任务、优先闭环验证、常见失败模式等），无机器人专用微调。
- **扩展能力**：
  - **工具自动改进**：智能体可以读取工具文档并测试新工具，反馈改进。
  - **通过文本反思学习**：提供文本形式的示例航点列表（类似演示），可显著提升性能（如 CC-Opus 从 77% 提升至 100%）。

## 6. 主要结论与发现

- **零样本可行性**：VIA 无需机器人专用微调，仅通过提示和通用工具即可解决多种桌面操作任务，最高成功率达 100%（CC-Fable 在 Rainbow 和 LIBERO-Goal 详细提示下）。
- **性能随模型强弱提升**：CC-Fable > CC-Opus；Codex-5.6-Sol 总体上优于 Codex-5.5（尤其精度任务）。说明 VIA 直接受益于基础模型能力的提升。
- **文本演示提升显著**：对较弱模型（CC-Opus）提供示例航点列表从 77% 提升至 100%，表明通过文本反馈学习的高潜力。
- **成本与效率**：每个成功操作平均耗资 $1.1-$36.1，Codex 更廉价（平均 $4.1-7.2 比 CC 的 $9.5-15.1）。工具调用次数随模型理性提升而优化（CC-Fable 在多数任务调用更少且成功率更高）。
- **局限仍存**：T-block 高精度任务成功率低（10-40%），受限于当前模型的空间定位和控制器精度。此外，系统仅适用于准静态任务，不适用于动态任务（如接球）。

## 7. 优点

- **创新性**：首次将机器人控制重新定义为“计算机使用”式智能体任务，避免了 VLA 微调的瓶颈（数据稀缺、计算限制、通用能力退化）。
- **简洁与通用**：工具集仅 16 个，界面直观，无需任务专用代码或感知基元；任何支持 MCP 和截图输入的智能体均可直接使用。
- **可扩展性**：随着通用智能体模型的进步，机器人控制能力自动提升（无需重新训练），兼容最新模型生态。
- **实用价值**：低成本产生演示数据（文本或截图→航点列表），可用于训练更快的策略；适用于可靠性优先的组装任务或强化学习环境重置。
- **实验设计清晰**：系统性对比不同模型、提示力度、任务难度，统计了成本、工具调用次数等实用指标。

## 8. 不足与局限

- **依赖前沿模型**：需要当前最强的基础模型（如 Fable 5、GPT-5.6-Sol），推理慢、成本高（一次成功回合可达数十美元），不适合大规模部署或实时控制。
- **计算基础设施**：需要多台 RGB-D 相机重建 3D 点云，并运行浏览器界面，系统复杂度较高。
- **任务范围有限**：仅测试了准静态桌面操作；动态任务、移动操作、接触力控制等未涉及。控制器仅为简单 PI 插值，无避碰或力反馈。
- **精度瓶颈**：高精度任务（T-block 允许 8mm 误差）成功率低，受限于模型对像素级位置的判断和控制器精度。
- **实验规模偏小**：每个配置仅 10 个种子，部分任务（T-block）低成功率下统计不确定度大；未与现有基线方法（如 VLA、CaP、扩散策略）进行同环境直接对比，无法证明 VIA 优于传统方法。
- **可重复性风险**：依赖于商业 API 和模型版本，不同时间点的智能体行为可能变化；论文未提供开源代码或界面复现细节。
- **提示工程敏感性**：提示词设计（尤其详细提示）需要人工干预，且对不同模型效果不同（CC-Opus 受益但 Codex-5.5 无改善），泛化性存疑。
