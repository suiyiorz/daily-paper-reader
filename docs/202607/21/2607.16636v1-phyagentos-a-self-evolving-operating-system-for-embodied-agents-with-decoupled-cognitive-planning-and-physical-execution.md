## 摘要
视觉-语言-动作模型、世界模型和智能体规划器各自推动了物理智能的发展，但它们的组合缺乏跨异构具身的通用执行抽象、共享状态、语义验证和持久经验。我们提出PhyAgentOS，一个运行时基础，将调度、验证、记忆、基准测试和安全作为系统级服务提供。其会话中心运行时将会话（而非单个动作）视为调度、兼容性预检、监督执行、证据收集和验收的最小单元。为了将认知与物理执行解耦，认知-物理边界是一个文件系统：状态即文件协议将跨层状态具体化为带有YAML的Markdown，生成可检查、可版本化的记录，而无需在智能体和运行时层之间产生代码依赖。这些视图形成了一个统一的认知状态空间，对齐意图、能力、环境、执行和经验。会话验证器通过基于证据的成功、失败或重新规划判定，区分执行终止与语义任务完成。验证结果通过认知记忆整合为可复用的知识和纠正经验，从而在不重新训练的情况下闭环试错循环。基准测试重复使用部署会话和验证路径，使结果可追溯至真实执行。分层安全约束策略驱动和智能体驱动的执行：包括预检、动作桥接、安全卫士、心跳监控和目标本地约束。验证是递进的：游戏测试认知规划，仿真增加动态和控制，真实机器人增加硬件噪声，而认知层保持不变。PhyAgentOS在Optimus-67、StarDojo和DST-Dojo上进行基准测试，在19种以上的仿真和物理具身上验证，并在LIBERO、Calvin和RoboCasa365上针对多种VLA模型取得了提升。

## Abstract
Vision-language-action models, world models, and agentic planners each advance physical intelligence, yet their composition lacks a common execution abstraction, shared state, semantic verification, and persistent experience across heterogeneous embodiments. We present PhyAgentOS, a runtime foundation delivering scheduling, verification, memory, benchmarking, and safety as system-level services. Its Session-Centered Runtime treats a session, not an action, as the minimum unit of scheduling, compatibility preflight, supervised execution, evidence collection, and acceptance. To decouple cognition from physical execution, the cognition-physics boundary is a file system: the State-as-a-File protocol materializes cross-layer state as Markdown with YAML, yielding inspectable, versionable records without code dependencies between Agent and Runtime layers. These views form a unified cognitive state space aligning intent, capabilities, environment, execution, and experience. The SessionVerifier distinguishes execution termination from semantic task completion via evidence-grounded verdicts of success, failure, or replan. Verified outcomes are consolidated through epistemic memory into reusable knowledge and corrective lessons, closing a trial-and-error loop without retraining. Benchmarking reuses the deployment session and verification path, so results trace to real execution. Layered safety constrains both policy-driven and agent-driven execution: preflight, action bridges, SafetyGuard, heartbeat monitoring, and target-local constraints. Validation is progressive: games test cognitive planning, simulation adds dynamics and control, real robots add hardware noise, with the cognitive layer held constant. PhyAgentOS is benchmarked on Optimus-67, StarDojo, and DST-Dojo, validated on 19+ simulated and physical embodiments, and gains on LIBERO, Calvin, and RoboCasa365 across multiple VLA models.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：当前具身智能领域存在三种主流范式——视觉-语言-动作模型（VLA）、世界模型和智能体系统——它们各自擅长一个方面，但简单的组合存在**结构性缺陷**：缺乏统一的执行抽象和共享状态表示，没有语义验证机制（例如：机械臂执行轨迹完成但未抓到杯子，系统仍报告成功），认知规划与物理执行耦合紧密，且缺乏跨会话的持久经验。现有的机器人中间件（如ROS）仅提供消息传递，并未提供任务级调度、验证、记忆和安全等系统级服务。
- **整体含义**：PhyAgentOS旨在成为一个**具身智能的操作系统**，位于现有模型之下，为物理智能体提供调度、验证、记忆、基准测试和安全作为一等系统服务，从而实现认知与物理的解耦、可追溯的执行和自我演化。

## 2. 方法论

### 核心思想
- **认知-物理边界视为文件系统**：所有跨进程数据（目标、技能、会话、观察、教训、知识）都通过**状态即文件（State-as-a-File）**协议具体化为人类可读的Markdown文件（嵌入YAML块），避免代码依赖。
- **会话为中心运行时（Session-Centered Runtime）**：将会话（而非单个动作）作为调度、预检、执行、监控和验收的最小单元。会话包含任务目标、选择的技能运行时、目标、前提条件、验收标准等。
- **智能体层与运行时层解耦**：智能体层负责决策（是什么），运行时层负责执行（如何做）。两者通过协议文件交互，无需导入对方实现。

### 关键技术细节
- **协议层文档**：核心协议包括`SESSIONS.md`（会话事务）、`SKILLRUNTIME.md`（能力定义）、`TARGETS.md`（目标描述）、`ENVIRONMENT.md`（环境感知）、`LESSONS.md`（经验记录）等，外部YAML文件管理感知、安全等参数。
- **WatchdogSupervisor**：会话调度器，负责认领会话、执行兼容性预检、创建SessionRunner、监控心跳、超时/取消传播、写回结果。
- **SessionVerifier**：基于证据的语义验证器，对执行结果进行`success`、`failure`或`replan`判定，区分执行终止与任务完成。
- **Epistemic Memory（认知记忆）**：分级记忆架构，包括情景记忆（原始会话）、语义记忆（聚合经验）、程序记忆（可复用技能）。正例存入`KNOWLEDGE.md`，失败经验存入`LESSONS.md`，闭环试错。
- **分层安全架构**：兼容性预检（Preflight）、动作桥（Action Bridge）、安全卫士（SafetyGuard）、心跳监控、目标本地约束，从不同阶段保障安全。
- **两种执行流**：**策略驱动**（PolicySkillRuntime，适用于VLA等连续控制模型）和**智能体驱动**（BuiltinSkillRuntime，适用于工具调用式步骤决策）。

## 6. 主要结论与发现

- PhyAgentOS作为一个自演化的具身智能操作系统，能够在不重新训练模型的情况下提升任务成功率。
- **Optimus-67**：接近饱和的性能（Wood 99%, Stone 96%），并在RedStone上达到30%（超越所有基线）。

- **DST-Dojo**：平均生存天数从1.02提高到2.10（+106%），达到30%的Day 3生存率。
- **LIBERO**：在4种策略上均有提升（+0.4至+1.3个百分点）。
- **CALVIN**：全链成功率提升显著，特别是π0的5/5从38.9%提升至45.6%（+6.7）。
- **RoboCasa365**：整体成功率提升7.2至9.2个百分点，共解救62个 episode。
- 这些改进源于**会话验证器**的语义判定和**认知记忆**的闭环经验积累，能够识别和纠正可恢复的执行错误。
