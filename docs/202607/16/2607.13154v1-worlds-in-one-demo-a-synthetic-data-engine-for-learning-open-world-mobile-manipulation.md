## 摘要
学习开放世界移动操作策略需要大量数据以实现空间泛化、长程鲁棒性和场景泛化。当前主流的数据收集范式，如遥操作和UMI，需要大量人力且成本高昂。为超越手动数据收集的规模限制，我们旨在通过可扩展的数据生成最大化每次人类演示的价值。为此，我们提出WANDA：通过综合数据引擎从单个演示学习开放世界移动操作。WANDA首先从源RGBD观测中重建背景高斯溅射和机器人与物体交互轨迹，作为后续规划和渲染的世界基底。然后，它将富含接触的机器人与物体交互片段重新排列成广泛的空间配置，利用全身运动规划将其链接成新轨迹。为增强长程鲁棒性，它应用校正状态扩展来增加移动操作不同阶段机器人和物体状态的多样性。为实现跨环境泛化，在从日常照片生成的不同三维世界上合成轨迹。此外，我们通过将渲染的机器人和物体网格与高斯溅射背景合成来生成照片级逼真的观测。我们在各种场景的大量仿真和真实世界任务中评估我们的方法。实验表明，使用WANDA训练的策略从单个真实演示中实现了长程鲁棒性、广泛的空间泛化和跨环境泛化。此外，WANDA自然支持跨实体数据生成，通过在另一个具有不同形态的移动操作器上进行零样本部署得到验证。

## Abstract
Learning open-world mobile manipulation policies requires vast data to achieve spatial generalization, long-horizon robustness, and scene generalization. Current prevailing data collection paradigms, teleoperation and UMI, demand prohibitive human effort and cost at scale. To scale beyond the limits of manual data collection, we seek to maximize the value of each human demonstration by scalable data generation. To this end, we introduce WANDA: learning open-World mobile mANipulation from one demonstration via a synthetic DAta engine. WANDA first reconstructs background Gaussian splats and robot-object interaction trajectories from source RGBD observations, as a world substrate for later planning and rendering. It then rearranges contact-rich robot-object interaction segments into extensive spatial configurations, utilizing whole-body motion planning to chain them into new trajectories. To enhance long-horizon robustness, it applies Corrective State Expansion to increase the robot and object state diversity at different stages of mobile manipulation. To unlock cross-environment generalization, trajectories are synthesized on diverse generated 3D worlds from everyday photos. Furthermore, we synthesize photo-realistic observations by compositing rendered robot and object meshes with Gaussian splatting backgrounds. We evaluate our approach on extensive simulation and real-world tasks in various scenes. Experiments show that policies trained with WANDA achieve long-horizon robustness, broad spatial generalization and cross-environment generalization from one real demonstration. Moreover, WANDA naturally supports cross-embodiment data generation, validated by zero-shot deployment on another mobile manipulator with a distinct morphology.

### 2. 论文提出的方法论

*   **核心思想**：
    WANDA 将一次人类演示分解并抽象为“世界基底”（背景）和“交互轨迹”（机器人-物体交互模式）。然后，它通过操作和重组这些组件，在新的、多样化的合成世界中生成新的轨迹和对应的视觉观测，实现“一镜一世界”式的数据爆炸。

*   **关键技术细节与流程**：
    WANDA 的数据生成流程（称为“Generative Mode”）包含以下四个步骤：
    1.  **场景与轨迹重建**:
        *   **世界基底重建**: 从一次单视角的RGB-D演示视频中，使用3D高斯泼溅（3D Gaussian Splatting, 3DGS）技术重建出静态背景的高质量3D场景表示。
        *   **交互轨迹重建**: 同时，通过视觉SLAM和手眼标定等技术，恢复出演示过程中机器人末端执行器和被操作物体的精确6-DoF轨迹。

    2.  **轨迹重组与重规划**:
        *   **接触片段分解**: 将重建出的长程交互轨迹，根据机器人与物体的接触状态（如接近、抓取、移动、放置），分解为几个关键的、富含接触信息的轨迹片段。
        *   **空间重配置**: 改变这些轨迹片段中的物体和机器人的初始/目标位置，创造大量新的空间配置。
        *   **全身运动规划**: 利用全身运动规划器（Whole-Body Motion Planner），将重配置后的轨迹片段平滑地链接成新的、可执行的完整长程轨迹，确保动力学可行性（特别是对移动基座和操作臂的协调控制）。

    3.  **状态多样性增强**:
        *   **校正状态扩展**: 为了提升长程任务的鲁棒性，该方法在规划出的新轨迹基础上，应用“校正状态扩展”。这意味着在轨迹的各个阶段（如移动中、抓取时），随机扰动机器人关节角度或物体位姿，然后使用运动规划器重新进行局部修正，从而生成大量“状态多样化”的变体轨迹。

    4.  **场景与观测合成**:
        *   **跨环境泛化**: 为了打破场景依赖，WANDA 可以从日常照片集中随机选择一张或多张照片，使用文本到3D生成模型（如Luma AI）或单图像3D重建方法快速生成一个全新的、风格各异的虚拟3D场景。然后将上述合成轨迹渲染在这个新场景中。
        *   **照片级真实感观测合成**: 最后，通过将渲染出的机器人、物体的网格与新场景的3DGS背景进行图像层级合成，生成高度逼真的RGB-D观测图像，供策略网络训练。

*   **公式或算法流程说明**：
    *   整个流程可以视为一个数据生成的“编译-合成”过程：
        *   **输入**: 单次RGB-D演示 (D)。
        *   **阶段1 (编译)**: 从D中提取并解耦出静态世界模型 (S) 和动态交互模块 (M: 包含轨迹片段和接触模式)。
        *   **阶段2 (合成)**: 对新场景 (S') 和参数 (θ: 新的空间配置、状态扰动)，调用 M 和运动规划器生成新轨迹 (T')，最后通过可微渲染器/合成器生成与新世界 S’ 对齐的新观测 (O')。
        *   **输出**: 大规模、多样化的训练数据集 {T‘, O’}。

### 8. 不足与局限

1.  **对演示质量的依赖**：整个流程的起点是单次高质量的RGBD演示。如果演示中存在严重的遮挡、运动模糊或物体交互失败，重建的轨迹和场景基底质量会显著下降，从而影响后续生成数据的质量。
2.  **场景交互的简化**：当前方法主要处理“抓取-移动-放置”这类点对点任务。对于更复杂的、需要多步、精细或力控的交互任务（如开门、装配、工具使用），其轨迹分解与重组的有效性尚待验证。
3.  **运动规划的计算开销**：生成每个新轨迹都需要调用全身运动规划器，这在高通量数据生成过程中可能成为计算瓶颈。虽然未提及，但大规模并行生成时可能需要大量算力。
4.  **模拟到真实的鸿沟**：尽管使用了逼真的3DGS背景合成，但机器人-物体的物理交互（如抓取时的接触、物体动力学）仍然依赖于模拟器或简单的运动学假设。真实世界中的物理交互变化（如物体重量、摩擦系数）可能无法被充分覆盖。
5.  **场景生成质量**：从照片生成的3D场景用于数据合成时，可能会产生一些“不连贯”或“不真实”的细节。虽然实验证明了有效性，但场景的质量可能仍达不到专为仿真设计的场景的水平，特别是在复杂光照和遮挡情况下。
6.  **与最先进的深度强化学习（DRL）方法对比缺失**：论文未与那些也在尝试用合成数据训练通用策略的DRL方法（如通过域随机化或世界模型）进行比较，其绝对性能上限尚不清晰。
