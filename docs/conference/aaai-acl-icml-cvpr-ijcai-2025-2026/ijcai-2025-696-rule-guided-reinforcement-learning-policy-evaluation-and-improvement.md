# 论文详细中文总结

## 1. 核心问题与整体含义（研究动机和背景）

深度强化学习（DRL）在游戏等领域取得了显著成就，但其决策缺乏可解释性，且难以有效利用领域知识来评估和改进策略。现有方法如可解释RL（XRL）多聚焦于事后解释，而策略评估主要依赖离线评估或基于环境的测试，缺乏从策略自身决策中提取符号知识并利用领域知识进行泛化评估的手段。本文旨在解决以下问题：如何利用领域知识以规则形式表达，用于评估深度RL策略的弱点（特别是在泛化方面的不足），并基于此改进策略性能。作者提出的LEGIBLE框架将规则挖掘、蜕变关系泛化与规则引导执行相结合，为RL策略的可解释评估与改进提供了新范式。

## 2. 方法论

LEGIBLE包含三个依次构建的步骤：

- **步骤1：规则挖掘（Rule Mining）**
  - 收集策略执行经验（状态-动作对），使用LIME计算每个特征对动作选择的重要性。
  - 对每个动作和极性（正/负），使用RIPPER算法（修改后结合FOIL信息增益和特征重要性）学习规则。规则形式为：`⊗action(a) ← ∧_{i∈I} fi = vi`，其中⊗∈{+,−}表示正规则（应采取a）或负规则（应避免a）。
  - 通过准确度（≥0.9）和覆盖率（≥0.01）阈值筛选规则。

- **步骤2：规则泛化（Rule Generalization）**
  - 引入蜕变关系（MRs）来描述环境中的对称性或特征关系。MR由一组特征关系组成，每个特征关系`⟨a, a′, i, j, R⟩`表示动作从a变为a′时特征i与j的值满足关系R。
  - 给定MR集合M，基于`ρ1 mr ρ2`的定义自动生成每个规则`ρ`的泛化集合`Mgen(ρ)`（包含原始规则和所有通过MR可推导出的规则）。

- **步骤3：规则引导执行（Rule-Guided Execution）**
  - 执行策略时，每步检查触发的泛化规则。若唯一正规则触发则强制执行；若多个正规则冲突或无正规则，则禁止所有负规则对应的动作（Q值设负无穷），然后选择Q值最高的动作。
  - 计算引导执行的平均累积奖励，与无引导的策略对比（使用Welch检验判断显著性），若显著更高则判定该泛化规则集揭示了策略弱点。
  - 通过贪心算法（Algorithm 2）组合多个有利的规则集，形成最终改进的策略。

## 3. 实验设计

- **环境与数据集**：共11个环境。
  - 6个PAC-Man关卡（small、small-nc、medium、medium-nc、original、original-nc），状态维度69或117，动作空间为5（四个方向+不动）。
  - 5个highway-env场景（highway、highway-fast、merge、intersection、roundabout），状态为ego车辆周围10辆车的7个属性。
- **训练配置**：使用DQN算法，PAC-Man环境训练2.5M或5M步，highway-env训练500K步。每个环境训练5个独立策略。
- **规则挖掘**：采样600个episode作为经验，使用LIME+RIPPER，规则筛选阈值：准确度0.9，覆盖率0.01。
- **基线方法**：
  - 随机测试（RT）：随机选择3%的状态，强制或禁止随机动作。
  - 随机规则（RR）：随机生成与MRs生成规则集大小和分布相似的规则，作为MRs的消融对照。
- **评估指标**：平均累积奖励及其标准误；弱点检测比率（显著优于无引导的次数占总评估次数的比例）。
- **评估episode数**：PAC-Man 250，highway-env 100。

## 6. 主要结论与发现

- LEGIBLE在所有11个环境中均能检测到策略弱点（弱点检测率5%~24%），且基于MRs的规则集（MT）平均优于随机测试（RT）和随机规则（RR）。
- 规则引导执行可显著提升策略累积奖励：PAC-Man环境中提升幅度平均约20%~50%，最高达273%（highway-fast环境）。
- 延长训练无法达到同等程度的提升，说明规则引导提供了扩展训练无法获得的泛化改善。
- 泛化规则解释了弱点的原因（例如策略学会了在某个方向避开墙壁，但未学会在其他方向同样避开）。

## 7. 优点

- **方法创新**：首次将蜕变测试（MT）引入RL策略评估，利用领域知识进行规则泛化，兼具可解释性和实用价值。
- **符号与神经结合**：通过规则挖掘从黑箱深度策略中提取部分符号表示，便于人类理解和推理。
- **评估与改进一体化**：规则引导执行既用于发现弱点，又可直接组合规则集形成改进策略，无需重新训练。
- **实验环境多样**：包含游戏（PAC-Man）和自动驾驶（highway-env）两类领域，具有一定代表性。
- **对比基线设计**：RT和RR分别控制随机干预和随机规则，有效隔离了MRs的贡献。

- **蜕变关系依赖人工定义**：MRs需要领域专家手动设计，限制了自动化和可扩展性，且可能无法覆盖所有有效泛化。
